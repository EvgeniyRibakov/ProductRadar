# Руководство по внесению вклада

Спасибо за интерес к проекту ProductRadar! Мы приветствуем ваш вклад.

## Как внести вклад

1. **Fork репозитория**
   - Создайте форк проекта на GitHub
   - Клонируйте свой форк локально

2. **Настройте окружение**
   ```bash
   git clone https://github.com/YOUR_USERNAME/ProductRadar.git
   cd ProductRadar
   poetry install --with dev
   ```

3. **Создайте ветку для изменений**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Внесите изменения**
   - Пишите понятный и читаемый код
   - Следуйте стилю кода проекта (используйте black, isort)
   - Добавьте тесты для новой функциональности
   - Обновите документацию при необходимости

5. **Запустите тесты**
   ```bash
   poetry run pytest tests/
   poetry run black src/ tests/
   poetry run isort src/ tests/
   poetry run flake8 src/ tests/
   ```

6. **Зафиксируйте изменения**
   ```bash
   git add .
   git commit -m "Описательное сообщение о ваших изменениях"
   ```

7. **Отправьте изменения**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Создайте Pull Request**
   - Перейдите на GitHub и создайте Pull Request
   - Опишите ваши изменения подробно
   - Ссылайтесь на связанные issues

## Стиль кода

- Используем **Black** для форматирования кода
- Используем **isort** для сортировки импортов
- Следуем **PEP 8**
- Документируем все публичные функции и классы

## Тестирование

- Покрывайте новый код тестами
- Все тесты должны проходить перед отправкой PR
- Стремитесь к покрытию кода тестами минимум 80%

## Сообщения коммитов

Используйте понятные сообщения коммитов:
- `feat: добавить новую функцию`
- `fix: исправить баг`
- `docs: обновить документацию`
- `test: добавить тесты`
- `refactor: рефакторинг кода`

## Вопросы?

Если у вас есть вопросы, создайте issue на GitHub или свяжитесь с мейнтейнерами.
