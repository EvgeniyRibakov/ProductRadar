# Анализ: LLM (GPT) vs Парсер для поиска трендовых товаров

**Дата:** 17 октября 2024
**Задача:** Мониторинг beauty продуктов в TikTok Shop для разработки новых продуктов

---

## Сравнительная таблица

| Параметр | Собственный парсер | LLM (GPT Deep Research) |
|----------|-------------------|--------------------------|
| **Время до запуска** | 4-5 недель | 0 дней (сразу) |
| **Стоимость разработки** | 120-160 часов | 0 часов |

## Выводы из исследования

### Подтверждённые преимущества LLM:

1. **Адаптивность к контенту**
   - Нейросети обучаются на разнообразных данных
   - Не требуют обновления при изменении структуры сайта
   - Понимают контекст и нюансы (натуральность, multi-benefit)

2. **Качество анализа**
   - Выявление скрытых закономерностей
   - Фильтрация нерелевантных продуктов
   - Оценка соответствия бренду

3. **Практическое применение**
   - Крупные компании (Amazon) используют для product research
   - Применяется в маркетинге для анализа трендов
   - Реальные кейсы генерации рекламных материалов

### Ограничения LLM:

1. **Частота использования**
   - Deep Research имеет ограничения по запросам
   - Не подходит для real-time мониторинга
   - Оптимально: 1-2 раза в неделю

2. **Зависимость от источников**
   - Качество результатов зависит от доступности данных
   - Некоторые метрики TikTok могут быть недоступны
   - Требуется ручная проверка ссылок

3. **Масштабируемость**
   - Меньший объем данных vs парсер (30-50 vs 100+)
   - Не подходит для сбора массивов данных

## Рекомендация

### Для задачи Likato: **LLM подход предпочтительнее**

**Обоснование:**
- Задача = найти качественные возможности, не собрать все данные
- 30-50 проверенных продуктов/неделю достаточно
- Экономия 120-160 часов разработки
- Старт сегодня vs через месяц

### Оптимальная стратегия:

**Фаза 1 (месяцы 1-3): Только LLM**
- Еженедельные исследования через ChatGPT Pro
- Накопление базы продуктов
- Валидация гипотез
- **Стоимость:** $20-30/мес
- **Время:** 1-2 ч/неделя

**Фаза 2 (после 3 месяцев): Оценка необходимости парсера**
- Если LLM достаточно → продолжить
- Если нужен больший объем → разработать парсер
- Возможна комбинация: LLM для анализа + парсер для сбора

## Риски и митигация

### Риски LLM подхода:

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Недостоверные данные | Средняя | Ручная проверка топ-10, требование ссылок |
| Ограничения API | Средняя | Использовать 1-2 раза/неделю, не чаще |
| Неполнота данных | Высокая | Комбинировать с ручным поиском |
| Изменение pricing | Низкая | Мониторить стоимость ChatGPT Pro |

### Риски парсера:

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Блокировки TikTok | Высокая | Ротация прокси ($30-50/мес) |
| Изменение структуры | Высокая | Регулярные обновления (5-10 ч/мес) |
| Поломка парсера | Средняя | Мониторинг, быстрое реагирование |
| Затраты времени | Высокая | Заложить 150+ часов разработки |

---

## Критерии выбора парсера (если понадобится)

Парсер стоит разрабатывать только если:
- ✅ Нужно >100 продуктов/неделю
- ✅ Требуется real-time мониторинг
- ✅ Необходимы специфичные метрики недоступные LLM
- ✅ Есть ресурсы на разработку и поддержку
- ✅ LLM показал недостаточную эффективность
